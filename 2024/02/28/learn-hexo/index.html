HEXO

安装 hexo

安装 git
安装 Node.js

安装 hexo

npm install -g hexo-cli

基本指令

新建blog目录

mkdir blog

进入目录

cd blog

初始化目录

创建 blog 文件夹并生成目录环境

hexo init blog

进入 blog 文件夹

cd blog

下载相关依赖

npm install

新建文章

hexo new "learn git"

生成静态文件

hexo generate

hexo g

选项 描述
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动
-b, –bail 生成过程中如果发生任何未处理的异常则抛出异常
-f, –force 强制重新生成文件
Hexo 引入了差分机制，如果 public 目录存在，那么 hexo g 只会重新生成改动的文件。
使用该参数的效果接近 hexo clean && hexo generate
-c, –concurrency 最大同时生成文件的数量，默认无限制

启动服务器

hexo server

默认情况下，访问网址为： http://localhost:4000/

选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

部署网站

hexo deploy

hexo d

选项 描述
-g, –generate 部署之前预先生成静态文件

清除缓存文件

hexo clean

清理包括db.json、public，更换主题，需要运行该命令

一键部署

安装 hexo-deployer-git

npm install hexo-deployer-git --save

修改配置文件 _config.yml

如果 repo: https://github.com/username/username.github.io，那么生成后的博客地址：username.github.io

如果 repo: https://github.com/username/自定义仓库，那么生成后的博客地址：username.github.io/自定义仓库

注意，如果推送的仓库非’username.github.io’，那么分支必须是填’gh-pages’

清除缓存并部署

hexo clean & hexo deploy

主题开发

标签 含义
<% 脚本标签，用于流程控制
<%_ 删除前面的空格符
<%= 输出数据到模板
<%- 输出非转义数据到模板
<%# 注释
<%% 输出字符串『<%』
>% 脚本结束标签
-%> 结束时，删除换行符
_%> 结束时，删除空格符
<%- include(“index.ejs”) %> 引入其他模板
变量 描述
site 总体变量，几乎都是从这里开始的
site.posts 所有文章
site.posts[0].path 文章路径，带日期的
site.posts[0].slug 文章路径，根据项目文件夹的路径来的
site.posts[0]._id 文章的唯一 id，后面会用于 active 对比
site.posts[0].title 文章的标题
site.posts[0].date 文章的时间
page.date 在直接访问文章路径下，文章的时间
page.title 在直接访问文章路径下，文章的标题
page._id 在直接访问文章路径下，文章的的唯一 id，后面会用于 active 对比
page.content 引入对应文章的正文
config.xxx 总体配置文件的引用 _config.yml
theme.xxx 主题配置文件 theme._config.yml
<%- body %> 同时引入 post.ejs 和 index.ejs
<%- css(path, …) %> 引入 css 文件
<%- js(path, …) %> 引入 js 文件
基本上，就用这两组标签，其他的也用不上。
